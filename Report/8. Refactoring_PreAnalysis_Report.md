## ECB(Entity-Control-Boundary) 분석 (요약)

### 역할 매핑

- `src/magic_square/logic.py`: **Control**
  - 규칙 실행/판정/풀이 절차(`solution`, `is_magic_square` 등)
  - Entity(데이터 전용 구조)는 별도 타입 없이 `list[list[int]]`로 직접 처리

- `src/magic_square/boundary.py`: **Boundary**
  - 입력 계약(I1~I4) 검증 + 에러 코드/메시지 매핑
  - 도메인 오류(`MagicSquareDomainError`)를 계약 오류(`MagicSquareContractError`)로 변환(일부)

- `src/magic_square/gui/main_window.py`: **UI Boundary(프레임워크 UI)**
  - UI 입력 수집 → boundary 호출 → 결과/오류 표시
  - UI 외 책임 혼재 후보: `_DEFAULT_GRID`(샘플 데이터), 결과 문자열 포맷 규칙

### 잘못된 위치로 보이는 코드 이동 제안(방향성)

- `gui/main_window.py`의 `_DEFAULT_GRID`:
  - UI 위젯 코드가 아닌 샘플/시드 데이터 성격
  - 이동 후보: `magic_square/constants.py` 또는 `magic_square/samples.py` 같은 별도 모듈

- `gui/main_window.py`의 결과 문자열 조립:
  - 도메인/경계로 이동하기보다는 UI 계층 내부에서 포맷터 함수로 분리하거나 UI 전용 프레젠터 모듈로 이동 권장
