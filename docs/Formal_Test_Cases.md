# Magic Square (4×4) — Formal Test Cases

**근거 원본:** 저장소 루트 `magic_square_tc.xlsx` (시트: 문서정보, 테스트케이스, 요약)  
**계약 원천:** `docs/MagicSquare_4x4_TDD.md` (FR-01~FR-05, §6 BR, §8 U-01~U-09, §9.1~9.2, §11 탐지 순서)  
**문서 목적:** 엑셀의 시나리오·우선순위를 유지하되, **대외 계약**과 불일치하는 기대값은 PRD 기준으로 정정한 **정식 테스트 케이스** 명세이다.

---

## 0. 계약 요약 (본 문서 전 TC 공통)

| 항목 | 규칙 |
|------|------|
| 좌표 | **1-index**, `1 ≤ r,c ≤ 4` (BR-06). 엑셀에 0-index 좌표가 있으면 본 문서에서 **PRD 좌표로 치환**한다. |
| 성공 출력 | 길이 6의 정수 배열 **`[r1,c1,n1,r2,c2,n2]`** — `n1`은 row-major **첫** 빈칸 값, `n2`는 **둘째** 빈칸 값 (FR-05, BR-14). |
| 입력 | FR-01 통과 전제: 4×4, **0 정확히 2개**, 나머지 `1~16` 유일. 위반 시 **첫 탐지 규칙**에 따른 `code`·`message` (§11, §8.1). |
| 도메인 실패 | 두 배치 모두 비마방진 → 경계에서 `NOT_MAGIC_SQUARE` / 고정 메시지 (§8.1 U-09). |
| `is_magic_square` 등 완성 판정 | **0이 없는** 완성 격자에만 적용 (FR-04). |

**엑셀 대비 정정 요약:** 빈칸 개수 ≠ 2인 입력에 대해 엑셀은 `find_blank_coords` 반환만 서술한 경우가 있으나, PRD에서는 **경계 검증 실패**(`BLANK_COUNT_INVALID` 등)가 맞다. `solution` 반환 형은 튜플 리스트가 아니라 **`int[6]`**이다.

## 4. 구현 체크리스트

1. pytest에서 **§8.1 `message` 바이트 단위 동일** assert.  
2. 성공 경로는 **`[r1,c1,n1,r2,c2,n2]`** 만 허용 (튜플 리스트 금지).  
3. 좌표 assert는 **항상 1-index**.  
4. `T-05`용 “두 시도 모두 실패” 픽스처는 §9.2 및 회귀 정책에 따라 **저장소에 고정**한다.
